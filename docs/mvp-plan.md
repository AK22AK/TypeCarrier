# MVP 计划

本文保留 TypeCarrier 早期 MVP 定义和阶段边界。后续功能规划统一维护在 [路线图](roadmap.md)。

## MVP 定义

当下面几件事成立时，MVP 就算成功：

1. iPhone app 能接收任意键盘输入的文本。
2. iPhone app 能把当前文本发送给 Mac app。
3. Mac app 作为菜单栏应用运行并接收文本。
4. Mac app 能把文本插入当前聚焦输入框。
5. 这个流程可以低摩擦地反复使用。

## 阶段 1：本地原型

构建最小可用的端到端闭环。

- 创建 iOS SwiftUI app，包含大文本编辑器和发送按钮。
- 创建 macOS 菜单栏 app。
- 建立本地设备发现和连接。
- 从 iOS 向 Mac 发送纯文本 payload。
- Mac 端写入剪贴板并模拟粘贴。
- 显示简单连接状态和发送状态。

## 阶段 2：可用性加固

让它足够可靠，能支撑日常自用。

- 增加 macOS Accessibility 首次授权引导。
- 增加重连行为。
- 增加发送失败重试。
- 尽量保存并恢复原剪贴板。
- 增加“仅复制到剪贴板，手动粘贴”的 fallback。
- 增加发送后的行为配置：清空文本、保留文本或全选文本。

## 阶段 3：产品化候选项

核心闭环足够顺手后，再按路线图推进产品化能力。

- 多设备连接与切换管理。
- 轻量发送历史。
- 本地加密和传输安全审查。
- Android / Windows 探索。
- TV / 盒子输入探索。
- iOS 小组件或 Shortcuts action。

## 明确推迟

- AI 转写。
- AI 改写。
- 文件传输。
- 剪贴板管理器 UI。
- 团队协作。
- 云账号。
