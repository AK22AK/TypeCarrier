# TypeCarrier 0.1 Beta 1

发布日期：待定

## 定位

0.1 Beta 1 是 TypeCarrier 的首个小范围测试版本，用来验证核心闭环：

> 在 iPhone 上输入或语音转文字，发送到 Mac 当前光标所在的位置。

这个版本适合自己试用、朋友试用或 GitHub 公开 Beta；不应被描述为成熟正式版。

## 运行要求

- iOS 26.0 或更新版本。
- macOS 26.0 或更新版本。
- iPhone 与 Mac 位于同一个局域网内；推荐先用 iPhone 热点场景验证。
- Mac 端需要授权辅助功能权限，才能把收到的文本插入当前输入焦点。

## 本版包含

- iPhone 输入和发送界面。
- macOS 菜单栏接收端。
- 基于 Multipeer Connectivity 的局域网发现和传输。
- Mac 端收到文本后写入剪贴板并模拟粘贴。
- 连接状态、接收状态和诊断入口。
- 成功发送后的输入区清空与焦点保留策略。

## 已知限制

- 当前按 1:1 receiver 约束工作：一个 Mac receiver 同一时间只应服务一个 iOS sender。
- 真机 iPhone 和 iOS Simulator 同时连接同一个 Mac 时，可能互相抢占连接。
- 不包含二维码配对、配对码、可信设备列表或多设备切换管理。
- 不包含云同步、互联网中转、Android、Windows 或 AI 转写能力。
- 自动粘贴依赖 macOS 当前焦点和辅助功能权限，失败时应优先检查 Mac 端状态和诊断日志。

## 发布前验证

发布前至少运行：

```sh
script/ci_sanity.sh
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierCore -destination 'platform=macOS' test
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrieriOS -destination 'generic/platform=iOS Simulator' build
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierMac -destination 'platform=macOS' build
```

建议额外做一次真机验证：

1. 启动 Mac 端 TypeCarrier，并确认辅助功能权限已授权。
2. 启动 iPhone 端 TypeCarrier。
3. 在 iPhone 输入中文、英文和换行文本。
4. 将 Mac 光标放到 TextEdit、浏览器输入框或聊天输入框。
5. 从 iPhone 发送，确认文本出现在 Mac 当前光标处。
6. 发送后确认 iPhone 输入区清空且仍可直接继续输入。
