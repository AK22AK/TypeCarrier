# TypeCarrier 0.1 Beta 1

[中文](0.1-beta.1.md)

Release date: 2026-05-17

## Positioning

0.1 Beta 1 is the first small-scope testing release of TypeCarrier. It validates the core loop:

> Type on the iPhone, or use iPhone speech-to-text, send the text to the current cursor position on the Mac.

This release is suitable for personal testing, friend testing, and GitHub technical preview. It should not be described as a mature stable release.

GitHub Release provides full source code, a macOS development testing package, and an Android debug APK for sideload testing. The iOS side is source-only and does not provide an installable GitHub build.

## Requirements

- iOS 26.0 or later.
- macOS 26.0 or later.
- iPhone and Mac on the same local network. The iPhone hotspot scenario is recommended for the first validation pass.
- Android 8.0 or later can sideload the Android APK. The Android side is still a sender testing build.
- The Mac app needs Accessibility permission to insert received text into the current input focus.

## Included

- iPhone input and send UI.
- Android input and send UI with manual connection, pairing code, and fast reconnect for paired Macs.
- macOS menu bar receiver.
- Local discovery and transport with Multipeer Connectivity.
- Android-to-Mac sender transport over local TCP.
- macOS text insertion through clipboard handoff and simulated paste.
- Connection state, receiver state, and diagnostics entry points.
- Input clearing and focus retention after successful send.

## Known Limitations

- Current behavior is a 1:1 receiver constraint: one Mac receiver should serve one iOS sender at a time.
- A real iPhone and an iOS Simulator connecting to the same Mac may contend for the receiver.
- The Android APK is a debug build for sideload testing only. It does not yet have release signing, Play distribution, or store release support.
- No QR pairing, pairing code, trusted device list, or multi-device switching.
- No cloud sync, internet relay, Windows, or AI transcription.
- The macOS zip is Apple Development / Personal Team signed. It is not a Developer ID notarized public distribution build, so Gatekeeper may block it.
- TestFlight, App Store distribution, and Developer ID notarized macOS builds require the paid Apple Developer Program and are not provided in this release.
- Automatic paste depends on the current macOS focus and Accessibility permission. If insertion fails, check the Mac receiver state and diagnostics first.

## Pre-Release Verification

Run at least:

```sh
script/ci_sanity.sh
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierCore -destination 'platform=macOS' test
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrieriOS -destination 'generic/platform=iOS Simulator' build
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierMac -destination 'platform=macOS' build
cd Apps/Android && ./gradlew testDebugUnitTest assembleDebug
```

Recommended device validation:

1. Launch TypeCarrier on the Mac and confirm Accessibility permission is granted.
2. Launch TypeCarrier on the iPhone.
3. Enter Chinese text, English text, and multiline text on the iPhone.
4. Put the Mac cursor in TextEdit, a browser input, or a chat input.
5. Send from the iPhone and confirm the text appears at the current Mac cursor.
6. Confirm the iPhone input area clears after send and remains ready for continued input.
7. Sideload the Android APK and complete first-time pairing with the same Mac pairing code.
8. Enter text or use speech-to-text on Android, send, and confirm the text appears at the current Mac cursor.
9. Restart the Android app and send again, confirming the paired Mac reconnects quickly without entering the pairing code again.
