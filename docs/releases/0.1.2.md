# TypeCarrier 0.1.2

[English](0.1.2.en.md)

发布日期：2026-06-16

## 定位

0.1.2 是 TypeCarrier 的第三个小范围测试版本，重点是让 iPhone / Android 到 Mac 的局域网发现、连接恢复和诊断提示更可靠。

这个版本适合自己试用、朋友试用或 GitHub 公开技术预览；不应被描述为成熟正式版。

GitHub Release 会提供完整源码、Android debug APK 侧载测试包，以及 Developer ID signed + notarized macOS DMG。iOS 端通过 TestFlight / App Store 路径分发，不在 GitHub Release 提供可安装包。

## 运行要求

- iOS 26.0 或更新版本。
- macOS 26.0 或更新版本。
- iPhone / Android 与 Mac 位于同一个局域网内；首次验证建议使用稳定 Wi-Fi 或 iPhone 热点场景。
- Android 8.0 或更新版本可侧载 Android APK。
- Mac 端需要授权辅助功能权限，才能把收到的文本插入当前输入焦点。

## 本版包含

- iPhone 输入和发送界面。
- Android 输入和发送界面，支持手动连接、配对码、已配对 Mac 快速重连和单目标自动连接。
- macOS 菜单栏接收端和主窗口入口。
- 基于 Multipeer Connectivity 的 iPhone -> Mac 局域网发现和传输。
- 基于局域网 TCP 的 Android -> Mac 发送通道。
- Mac 端收到文本后写入本地历史，并按当前配置执行粘贴。
- 连接状态、接收状态、粘贴诊断、连接自检和失败解释。

## 0.1.2 变化

- 强化 Android NSD 发现：服务启动失败会自动重试，服务解析失败会进行有界重试。
- Android 回到前台后会刷新发现，或对已信任的当前 Mac 自动重连。
- Android 发现单台未配对 Mac 时会自动选中为配对目标，但不会跳过配对码直接连接。
- Android / iOS / macOS 诊断界面增加连接自检提示，明确网络、配对、权限和连接失败原因。
- iPhone / Mac 的 Multipeer 搜索超时后会刷新发现，而不是直接停止到 idle。
- macOS Android bridge listener 失败后会进行有界自动重启。
- GitHub Release workflow 支持生成并上传 Developer ID signed + notarized macOS DMG。
- 将三端对外版本号统一为 0.1.2。

## 已知限制

- Android APK 是 debug 构建，仅用于侧载测试；还没有 release 签名、Play 分发或商店上架能力。
- iOS 端不提供 GitHub 可安装包；TestFlight / App Store 是 iOS 分发路径。
- macOS DMG 依赖 release workflow 的 Developer ID 和 notarization 配置。
- 不包含云同步、互联网中转、Windows 或 AI 转写能力。
- 多设备并发、跨 iPhone / Android sender 优先级和队列策略仍是后续能力。
- 自动粘贴依赖 macOS 当前焦点和辅助功能权限，失败时应优先检查 Mac 端状态和诊断日志。

## 发布前验证

发布前至少运行：

```sh
script/ci_sanity.sh
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierCore -destination 'platform=macOS' test
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrieriOS -destination 'generic/platform=iOS Simulator' build
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierMac -destination 'platform=macOS' build
cd Apps/Android && ./gradlew testDebugUnitTest assembleDebug
```

建议额外做一次真机验证：

1. 启动 Mac 端 TypeCarrier，并确认辅助功能权限已授权。
2. 启动 iPhone 端 TypeCarrier，通过 TestFlight 或本地安装覆盖到 0.1.2。
3. 在 iPhone 输入中文、英文和换行文本。
4. 将 Mac 光标放到 TextEdit、浏览器输入框或聊天输入框。
5. 从 iPhone 发送，确认文本出现在 Mac 当前光标处。
6. 侧载 Android 0.1.2 APK 后，使用同一台 Mac 的配对码完成首次配对。
7. 从 Android 输入或语音转文字并发送，确认文本出现在 Mac 当前光标处。
8. 重启 Android 应用或从后台切回前台，确认发现刷新和已配对 Mac 重连行为正常。
9. 发布后下载 GitHub Release 的 macOS DMG，挂载并确认 app bundle 版本为 0.1.2。
