# TypeCarrier 0.1.3

[中文](0.1.3.md)

Release date: 2026-07-05

## Positioning

0.1.3 is the fourth small-scope testing release of TypeCarrier. It focuses on send-with-Return, Android / iOS send experience alignment, and connection stability when Debug and Release Mac receivers coexist.

This release is suitable for personal testing, friend testing, and GitHub technical preview. It should not be described as a mature stable release.

GitHub Release provides full source code, an Android release-signed APK for sideload testing, and a Developer ID signed + notarized macOS DMG. iOS distribution uses the TestFlight / App Store path and does not provide an installable GitHub build.

## Requirements

- iOS 26.0 or later.
- macOS 26.0 or later.
- iPhone / Android and Mac on the same local network. A stable Wi-Fi network or iPhone hotspot is recommended for the first validation pass.
- Android 8.0 or later can sideload the Android APK.
- The Mac app needs Accessibility permission to insert received text into the current input focus.

## Included

- iPhone input and send UI with an optional send-with-Return mode.
- Android input and send UI with an optional send-with-Return mode, manual connection, pairing code, fast reconnect for paired Macs, and single-target auto-connect.
- macOS menu bar receiver and main-window entry point.
- iPhone-to-Mac local discovery and transport with Multipeer Connectivity.
- Android-to-Mac sender transport over local TCP.
- macOS local receive history and configurable paste handling with optional Return.
- Connection state, receiver state, paste diagnostics, connection self-checks, and failure explanations.

## Changes In 0.1.3

- Adds a send mode selector on iOS and Android: plain send or send followed by a Mac Return key press.
- Extends the Core payload protocol with an optional post-paste action while keeping legacy payloads as plain sends.
- Lets the Mac receiver press Return after successful paste when requested, and adds a local function-test page for Accessibility paste and Return validation.
- Aligns the Android send UI with iOS by placing the mode selector inside the send button instead of splitting the action into two send buttons.
- Gives macOS Debug builds a distinct bundle identifier and display name so they do not share Accessibility permission records with the published Release build.
- Stabilizes iOS / macOS receiver matching around durable receiver identity to reduce stale-discovery matches after Mac receiver restarts.
- Adds receiver bundle and variant metadata to Android / macOS discovery so Debug and Release Mac receivers can be distinguished.
- Simplifies Mac connection management so the Mac only provides the receiver pairing code, while iOS or Android senders initiate pairing.
- Unifies the user-facing version across all three platforms as 0.1.3.

## Known Limitations

- The Android APK is a release-signed build for GitHub sideload testing. It does not yet have Play distribution or store release support.
- The iOS side does not provide an installable GitHub build. TestFlight / App Store is the iOS distribution path.
- The macOS DMG depends on the release workflow's Developer ID and notarization configuration.
- No cloud sync, internet relay, Windows, or AI transcription.
- Multi-device concurrency, cross-iPhone / Android sender priority, and queueing remain future work.
- Automatic paste and Return depend on the current macOS focus and Accessibility permission. If insertion fails, check the Mac receiver state, function-test page, and diagnostics first.

## Pre-Release Verification

Run at least:

```sh
script/ci_sanity.sh
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierCore -destination 'platform=macOS' test
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrieriOS -destination 'generic/platform=iOS Simulator' build
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierMac -destination 'platform=macOS' build
cd Apps/Android && ./gradlew testDebugUnitTest assembleRelease
```

Recommended device validation:

1. Launch TypeCarrier on the Mac and confirm Accessibility permission is granted.
2. Launch TypeCarrier on the iPhone through TestFlight or a local 0.1.3 install.
3. Enter Chinese text, English text, and multiline text on the iPhone, then validate both plain send and send-with-Return.
4. Put the Mac cursor in TextEdit, a browser input, or a chat input, and confirm sent text appears at the current cursor while send-with-Return inserts a newline or submits.
5. Sideload the Android 0.1.3 APK and complete first-time pairing with the same Mac pairing code.
6. Enter text or use speech-to-text on Android, then validate both plain send and send-with-Return.
7. Install Mac Release and Debug builds side by side, confirm they appear with distinct names in the Accessibility list, and verify paste works after each is granted permission.
8. Use the Mac function-test page to run local paste-only and paste-with-Return checks.
9. After publishing, download the macOS DMG from GitHub Release, mount it, and confirm the app bundle version is 0.1.3.
