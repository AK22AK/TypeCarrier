# TypeCarrier 0.1.2

[中文](0.1.2.md)

Release date: 2026-06-16

## Positioning

0.1.2 is the third small-scope testing release of TypeCarrier. It focuses on more reliable local discovery, connection recovery, and diagnostics for the iPhone / Android to Mac loop.

This release is suitable for personal testing, friend testing, and GitHub technical preview. It should not be described as a mature stable release.

GitHub Release provides full source code, an Android debug APK for sideload testing, and a Developer ID signed + notarized macOS DMG. iOS distribution uses the TestFlight / App Store path and does not provide an installable GitHub build.

## Requirements

- iOS 26.0 or later.
- macOS 26.0 or later.
- iPhone / Android and Mac on the same local network. A stable Wi-Fi network or iPhone hotspot is recommended for the first validation pass.
- Android 8.0 or later can sideload the Android APK.
- The Mac app needs Accessibility permission to insert received text into the current input focus.

## Included

- iPhone input and send UI.
- Android input and send UI with manual connection, pairing code, fast reconnect for paired Macs, and single-target auto-connect.
- macOS menu bar receiver and main-window entry point.
- iPhone-to-Mac local discovery and transport with Multipeer Connectivity.
- Android-to-Mac sender transport over local TCP.
- macOS local receive history and configurable paste handling.
- Connection state, receiver state, paste diagnostics, connection self-checks, and failure explanations.

## Changes In 0.1.2

- Strengthens Android NSD discovery with automatic restart after service-start failures and bounded retries after resolve failures.
- Refreshes Android discovery when the app returns to the foreground, or reconnects to the selected trusted Mac.
- Auto-selects a single discovered unpaired Mac as the pairing target without skipping pairing-code validation.
- Adds connection self-check guidance to Android, iOS, and macOS diagnostics surfaces for network, pairing, permission, and connection failures.
- Refreshes Multipeer discovery after iPhone / Mac search timeouts instead of dropping senders back to idle.
- Restarts the macOS Android bridge listener after bounded listener failures.
- Adds GitHub Release workflow support for building and uploading a Developer ID signed + notarized macOS DMG.
- Unifies the user-facing version across all three platforms as 0.1.2.

## Known Limitations

- The Android APK is a debug build for sideload testing only. It does not yet have release signing, Play distribution, or store release support.
- The iOS side does not provide an installable GitHub build. TestFlight / App Store is the iOS distribution path.
- The macOS DMG depends on the release workflow's Developer ID and notarization configuration.
- No cloud sync, internet relay, Windows, or AI transcription.
- Multi-device concurrency, cross-iPhone / Android sender priority, and queueing remain future work.
- Automatic paste depends on the current macOS focus and Accessibility permission. If insertion fails, check the Mac receiver state and diagnostics first.

## Pre-Release Verification

Run at least:

```sh
script/ci_sanity.sh
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierCore -destination 'platform=macOS' test
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrieriOS -destination 'generic/platform=iOS Simulator' build
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierMac -destination 'platform=macOS' build
cd Apps/Android && ./gradlew testDebugUnitTest assembleDebug
```

Recommended device validation:

1. Launch TypeCarrier on the Mac and confirm Accessibility permission is granted.
2. Launch TypeCarrier on the iPhone through TestFlight or a local 0.1.2 install.
3. Enter Chinese text, English text, and multiline text on the iPhone.
4. Put the Mac cursor in TextEdit, a browser input, or a chat input.
5. Send from the iPhone and confirm the text appears at the current Mac cursor.
6. Sideload the Android 0.1.2 APK and complete first-time pairing with the same Mac pairing code.
7. Enter text or use speech-to-text on Android, send, and confirm the text appears at the current Mac cursor.
8. Restart the Android app or bring it back from the background, confirming discovery refresh and paired-Mac reconnect behavior.
9. After publishing, download the macOS DMG from GitHub Release, mount it, and confirm the app bundle version is 0.1.2.
