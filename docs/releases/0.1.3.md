# TypeCarrier 0.1.3

[English](0.1.3.en.md)

发布日期：2026-07-05

## 定位

0.1.3 是 TypeCarrier 的第四个小范围测试版本，重点是发送后回车、Android / iOS 发送体验对齐，以及 Debug / Release Mac 接收端共存时的连接稳定性。

这个版本适合自己试用、朋友试用或 GitHub 公开技术预览；不应被描述为成熟正式版。

GitHub Release 会提供完整源码、Android release-signed APK 侧载测试包，以及 Developer ID signed + notarized macOS DMG。iOS 端通过 TestFlight / App Store 路径分发，不在 GitHub Release 提供可安装包。

## 运行要求

- iOS 26.0 或更新版本。
- macOS 26.0 或更新版本。
- iPhone / Android 与 Mac 位于同一个局域网内；首次验证建议使用稳定 Wi-Fi 或 iPhone 热点场景。
- Android 8.0 或更新版本可侧载 Android APK。
- Mac 端需要授权辅助功能权限，才能把收到的文本插入当前输入焦点。

## 本版包含

- iPhone 输入和发送界面，支持可选的发送后回车模式。
- Android 输入和发送界面，支持可选的发送后回车模式、手动连接、配对码、已配对 Mac 快速重连和单目标自动连接。
- macOS 菜单栏接收端和主窗口入口。
- 基于 Multipeer Connectivity 的 iPhone -> Mac 局域网发现和传输。
- 基于局域网 TCP 的 Android -> Mac 发送通道。
- Mac 端收到文本后写入本地历史，并按当前配置执行粘贴和可选回车。
- 连接状态、接收状态、粘贴诊断、连接自检和失败解释。

## 0.1.3 变化

- iOS 和 Android 增加发送模式选择：普通发送或发送后让 Mac 按一次 Return。
- Core 协议增加可选的 post-paste action 字段，旧 payload 继续按普通发送处理。
- Mac 接收端在粘贴成功后可按 payload 要求发送 Return，并在功能测试页提供本机辅助功能粘贴和回车验证。
- Android 发送界面与 iOS 对齐，在发送按钮内提供下拉选择入口，而不是拆成两个发送按钮。
- Mac Debug build 使用独立 bundle id 和显示名称，避免和已发布 Release build 共用辅助功能授权记录。
- iOS / macOS 连接匹配逻辑使用稳定 receiver identity，降低 Mac 接收端重启后匹配到旧发现记录的概率。
- Android / macOS 发现逻辑增加接收端 bundle 和 variant 元数据，使 Debug 和 Release Mac 接收端可区分。
- Mac 端连接管理简化为只提供接收端匹配码，连接由 iOS 或 Android 发送端发起。
- 将三端对外版本号统一为 0.1.3。

## 已知限制

- Android APK 是 release 签名构建，用于 GitHub 侧载测试；还没有 Play 分发或商店上架能力。
- iOS 端不提供 GitHub 可安装包；TestFlight / App Store 是 iOS 分发路径。
- macOS DMG 依赖 release workflow 的 Developer ID 和 notarization 配置。
- 不包含云同步、互联网中转、Windows 或 AI 转写能力。
- 多设备并发、跨 iPhone / Android sender 优先级和队列策略仍是后续能力。
- 自动粘贴和回车依赖 macOS 当前焦点和辅助功能权限，失败时应优先检查 Mac 端状态、功能测试页和诊断日志。

## 发布前验证

发布前至少运行：

```sh
script/ci_sanity.sh
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierCore -destination 'platform=macOS' test
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrieriOS -destination 'generic/platform=iOS Simulator' build
xcodebuild -project TypeCarrier.xcodeproj -scheme TypeCarrierMac -destination 'platform=macOS' build
cd Apps/Android && ./gradlew testDebugUnitTest assembleRelease
```

建议额外做一次真机验证：

1. 启动 Mac 端 TypeCarrier，并确认辅助功能权限已授权。
2. 启动 iPhone 端 TypeCarrier，通过 TestFlight 或本地安装覆盖到 0.1.3。
3. 在 iPhone 输入中文、英文和换行文本，分别验证普通发送和发送后回车。
4. 将 Mac 光标放到 TextEdit、浏览器输入框或聊天输入框，确认文本出现在当前光标处，发送后回车会触发换行或提交。
5. 侧载 Android 0.1.3 APK 后，使用同一台 Mac 的配对码完成首次配对。
6. 从 Android 输入或语音转文字并发送，分别验证普通发送和发送后回车。
7. 同时安装 Mac Release 和 Debug build 时，确认系统辅助功能列表中显示不同应用名称，并分别授权后可正常粘贴。
8. 在 Mac 功能测试页执行仅粘贴和粘贴后回车测试。
9. 发布后下载 GitHub Release 的 macOS DMG，挂载并确认 app bundle 版本为 0.1.3。
