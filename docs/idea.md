# TypeCarrier Idea

## Problem

When working on a Mac, it is often useful to enter longer text by speaking into a phone. The phone is naturally closer to the mouth, easier to hold, and can use mobile input methods with better dictation quality.

The current manual workflow is:

1. Open Notes or another text field on iPhone.
2. Dictate with a preferred input method, such as Doubao keyboard.
3. Select all text.
4. Copy it.
5. Move back to Mac.
6. Paste into the active input field.

The friction is not speech recognition. The friction is moving recognized text from the phone to the Mac cursor.

## Core Idea

TypeCarrier turns the iPhone into a temporary text input panel for the Mac.

The user opens the iPhone app, types or dictates a text draft using any installed keyboard, then taps send. The Mac app receives the text and inserts it into the current focused input field.

## Product Positioning

TypeCarrier is not:

- A speech recognition engine.
- A clipboard history manager.
- A general file transfer tool.
- A full remote keyboard or mouse controller.
- A cloud sync platform.

TypeCarrier is:

- A focused text carrier from iPhone to Mac.
- A one-action replacement for manual copy and paste.
- A local-first utility for getting phone-entered text into the Mac cursor.

## Naming

The working product name is `TypeCarrier`.

The name suggests a small carrier for typed or dictated words. It avoids overly direct names such as `PasteToMac` or `Clipboard Sync`, while still pointing toward input and delivery.

Possible tagline:

> Carry words from your iPhone to your Mac cursor.
